Organizar Regras com Profiles

Voltar para Formas de Utilização

Objetivo

Profiles ajudam a centralizar regras de mapeamento em classes dedicadas. Esse formato reduz dispersão de configuração e facilita manutenção quando o projeto tem vários pares de tipos e transformações específicas.

Exemplo

public class UserProfile : MirrorProfile
{
    public override void Configure(IMirrorProfileExpression expression)
    {
        expression.CreateReflection<User, UserDto>()
            .ForMember<User, UserDto, string>(
                dto => dto.Name,
                user => $"{user.Name} - ACTIVE"
            );
    }
}

Vantagens

  • melhor organização das regras
  • facilidade de descoberta por assembly
  • separação entre pipeline de mapeamento e código de aplicação
