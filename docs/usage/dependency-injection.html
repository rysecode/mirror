Consumir o Mirror via Injeção de Dependência

Voltar para Formas de Utilização

Registro

builder.Services.AddMirror(config =>
{
    config.IgnoreNullValues = true;
    config.MaxDepth = 10;
});

Consumo

public class UserService
{
    private readonly IMirror _mirror;

    public UserService(IMirror mirror)
    {
        _mirror = mirror;
    }

    public UserDto Execute(User user)
    {
        return _mirror.Reflect<User, UserDto>(user);
    }
}

Quando Preferir DI

  • quando a aplicação já usa container em toda a arquitetura
  • quando a configuração do Mirror deve ser centralizada
  • quando profiles e regras globais precisam ser aplicados de forma uniforme
