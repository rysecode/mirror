Operações Seguras e Tratamento de Erros

Voltar para Formas de Utilização

Uso de ReflectSafe

try
{
    var result = source.ReflectSafe<Destination>();
}
catch (MirrorException ex)
{
    Console.WriteLine(ex.Message);
    Console.WriteLine(ex.Path);
    Console.WriteLine(ex.Stage);
}

Os métodos seguros ajudam a encapsular falhas de mapeamento dentro de MirrorException, tornando o tratamento mais previsível na aplicação e muito mais rico em contexto para suporte e observabilidade.

O que Agora Fica Disponível

  • o caminho completo da falha no grafo por meio de Path
  • a etapa do pipeline em Stage
  • o membro envolvido em MemberName
  • os tipos de origem e destino, incluindo tipos de membro
  • o valor que estava sendo processado quando a falha aconteceu

Exemplos de Caminho

  • Endereco.Cidade
  • Contatos[0]
  • Itens[principal]
  • ContatosPorCategoria[principal][0]

Quando Escrever Testes Específicos

  • quando há factories com lógica de domínio
  • quando coleções e dicionários se misturam em grafos profundos
  • quando existem regras contextuais de ignore
  • quando o projeto depende de preservação exata de estado em objetos existentes
  • quando o time precisa garantir mensagens de erro claras para suporte operacional
