Pré-requisitos

  • SDK .NET compatível com a versão alvo do pacote
  • um projeto de aplicação, biblioteca ou teste que fará o consumo do Mirror
  • referência ao namespace da biblioteca nos arquivos onde o mapeamento será utilizado

Instalação via .NET CLI

Essa é a forma mais simples para adicionar o pacote diretamente pelo terminal.

dotnet add package Mirror

Depois da instalação, o pacote passa a ser resolvido automaticamente pelo projeto, permitindo o uso de IMirror, MirrorConfiguration, extensions e integração com DI.

Instalação via PackageReference

Em cenários controlados por edição direta do arquivo de projeto, você pode adicionar a referência manualmente no .csproj.

<ItemGroup>
  <PackageReference Include="Mirror" Version="1.1.0" />
</ItemGroup>

Esse formato é útil quando a equipe centraliza gerenciamento de dependências via repositório ou arquivos de build.

Namespaces Comuns

using Mirror;
using Mirror.Extensions;
using Mirror.DependencyInjection;

Em geral, Mirror é usado para a API principal, Mirror.Extensions para métodos fluentes e Mirror.DependencyInjection para registro no container.

Verificando a Instalação

using Mirror;

var mirror = new Mirror.Mirror();

var source = new User
{
    Id = 1,
    Name = "Ana"
};

var dto = mirror.Reflect<User, UserDto>(source);

Se o projeto compilar e o objeto de destino for criado corretamente, a biblioteca está pronta para uso.
