Visão Geral

A implementação do Mirror foi organizada para fornecer um fluxo de mapeamento coerente tanto para casos simples quanto para cenários profundos. A biblioteca usa reflexão sobre propriedades públicas de instância, aplica regras configuradas e decide automaticamente como materializar o destino com base no tipo analisado.

Essa seção documenta não apenas o que a API faz, mas também como o comportamento é decidido internamente. Isso ajuda na manutenção da biblioteca, no diagnóstico de comportamento inesperado e na definição de novos recursos.

Resumo do Pipeline

  1. a configuração ativa é avaliada
  2. o tipo de destino é analisado
  3. propriedades ignoradas são filtradas
  4. transformações e factories são verificadas
  5. valores simples são copiados diretamente
  6. coleções, mapas e objetos complexos seguem pipelines dedicados
  7. limites como IgnoreNullValues, MaxDepth e proteção contra ciclos são respeitados
