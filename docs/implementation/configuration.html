MirrorConfiguration

MirrorConfiguration concentra o comportamento customizável da biblioteca. É nesse objeto que são registradas transformações, factories, limites e preferências globais de mapeamento.

var configuration = new MirrorConfiguration
{
    IgnoreNullValues = true,
    MaxDepth = 10
};

var mirror = new Mirror.Mirror(configuration);

Transformações por Membro

configuration.CreateReflection<User, UserDto>()
    .ForMember(dto => dto.Name, user => user.Name.ToUpperInvariant());

Transformações são úteis quando o destino precisa de um valor calculado a partir do objeto de origem.

Factories

configuration.AddFactory<CreateCompanyRequest, Company>(request =>
    Company.Create(request.Name, request.Document)
);

Factories ajudam a preservar invariantes de domínio e controlar a criação do destino.

IgnoreNullValues e MaxDepth

Quando IgnoreNullValues está habilitado, valores nulos da origem não sobrescrevem o destino. Já MaxDepth limita a profundidade de navegação para proteger o mapeamento em grafos extensos ou cíclicos.
