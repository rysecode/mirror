Tratamento de Exceções e Limites

Voltar para Implementação

MirrorException

try
{
    var dto = source.ReflectSafe<Destination>();
}
catch (MirrorException ex)
{
    Console.WriteLine(ex.Message);
    Console.WriteLine(ex.Path);
    Console.WriteLine(ex.Stage);
    Console.WriteLine(ex.MemberName);
    Console.WriteLine(ex.InnerException?.Message);
}

Falhas de mapeamento são encapsuladas em MirrorException, preservando a exceção original e acrescentando contexto suficiente para localizar com precisão onde o problema aconteceu.

A exceção expõe propriedades como Stage, Path, SourceType, DestinationType, MemberName, SourceMemberType, DestinationMemberType e SourceValue.

Detalhamento por Caminho

O caminho do erro acompanha a navegação do grafo. Isso significa que o diagnóstico deixa claro se a falha ocorreu em uma propriedade simples, item de coleção, valor de dicionário ou objeto aninhado.

  • Endereco.Cidade para objeto aninhado
  • Contatos[0] para item de lista
  • ContatosPorCategoria[principal][0] para listas dentro de dicionário
  • Itens[principal] para valor de dicionário

Exemplo de Mensagem

Erro ao mapear a propriedade 'Cidade' em 'Cliente.Endereco.Cidade'.
Origem: EnderecoDto.Cidade (String).
Destino: Endereco.Cidade (Int32).
Etapa: PropertyMapping.
Valor atual: "São Paulo".

Proteção contra Recursão Excessiva

Grafos complexos podem conter profundidade elevada ou ciclos. Por isso, a implementação respeita MaxDepth e mantém controle do contexto durante a recursão.

Cuidados Práticos

  • configure MaxDepth de forma intencional
  • use IgnoreNullValues em updates parciais
  • para construções de domínio complexas, prefira factories
  • escreva testes específicos para cenários ambíguos do projeto
  • em fluxos críticos, registre também Path e Stage no log da aplicação
