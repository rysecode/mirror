Suporte a Coleções

O Mirror consegue materializar diferentes famílias de coleções. Em vez de tratar apenas List<T>, a biblioteca identifica interfaces e tipos concretos compatíveis, criando a estrutura adequada para o destino sempre que possível.

  • List<T>
  • ICollection<T>
  • IList<T>
  • IReadOnlyList<T>
  • IReadOnlyCollection<T>
  • Collection<T>
  • ObservableCollection<T>
  • ReadOnlyCollection<T>
  • LinkedList<T>
  • Queue<T>
  • Stack<T>
  • ImmutableList<T>
  • ConcurrentBag<T>
  • arrays

Suporte a Mapas Chave-Valor

Para estruturas de mapa, o Mirror percorre pares Key/Value, preserva a chave e aplica o pipeline de mapeamento sobre o valor.

  • Dictionary<TKey, TValue>
  • SortedList<TKey, TValue>
  • IReadOnlyDictionary<TKey, TValue>
  • ReadOnlyDictionary<TKey, TValue>
  • ConcurrentDictionary<TKey, TValue>
  • ImmutableDictionary<TKey, TValue>
  • Hashtable
  • IDictionary

Profundidade em Itens Aninhados

O mesmo pipeline é reaplicado dentro de coleções e dicionários. Isso permite listas com objetos complexos, mapas com listas como valor e grafos híbridos aninhados em múltiplos níveis.
